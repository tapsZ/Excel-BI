## The three days

### [Day 1 — Excel Foundations](datasets/day1_excel_foundations/README.md)
*Getting comfortable and productive in Excel*

Eight modules taking you from the very basics — cells, formatting, formulas — through to functions, sorting and filtering, charts, and PivotTables. Ends with a 30-minute homework on a fresh dataset.

**Start here if you're new to Excel.**

### [Day 2 — Advanced Excel & Transitioning to Power BI](datasets/day2_advanced_excel_and_power_bi/README.md)
*Finishing Excel, then making the leap*

The morning finishes Excel properly: `XLOOKUP`, `INDEX`/`MATCH`, dynamic arrays, Power Query and Power Pivot. The afternoon introduces Power BI — and you'll find it familiar, because the engine underneath is the one you just learned.

**Start here if you already know Excel well.**

### [Day 3 — Visualizing, Publishing & Automating](datasets/day3_visualizing_publishing_automating/README.md)
*From data to decisions*

Choosing charts that don't mislead, telling a story with data, publishing and scheduling refreshes — then a capstone: build and present an executive dashboard from 23 sheets of messy real-world data.

## What you'll need

Please check this **before Day 2** — it's the one thing that catches people out.

| Day | What you need |
|---|---|
| **Day 1** | Excel — any recent version. Windows, Mac or the web version all work. |
| **Day 2 morning** | Excel with Power Query (built into Excel 2016 and later). *Power Pivot is Windows-only.* |
| **Day 2 afternoon, Day 3** | **Power BI Desktop — Windows only.** It's a free download from Microsoft. |
| **Day 3 publishing** | A Power BI account (free to sign up; sharing with others needs a Pro licence). |

> **Working on a Mac?** Day 1 and Day 2's morning run perfectly. Power BI Desktop doesn't exist for Mac, so for the afternoons you'll need Windows — a virtual machine, Boot Camp, or a shared Windows PC. If none of those is possible, you can still follow the lessons and open the finished example files, and nothing is wasted: Power Query and DAX work identically in both products.
>
> **A note on Excel versions.** Some newer functions (`XLOOKUP`, `IFS`, `FILTER`, `UNIQUE`, `SORT`) need Microsoft 365 or Excel 2021+. If you see `#NAME?` when you try one, your version is older — every lesson that uses them gives an `INDEX`/`MATCH` alternative that works everywhere.

## A running example

One dataset carries through all three days — **NovaTech Retail**, a consumer electronics business. You'll watch the same data grow from a plain list, into a formatted report, into a PivotTable, into a Power BI model, and finally into an executive dashboard.

That's deliberate. Each day you're not learning a new dataset, just a better thing to do with the one you already understand.

---

## The three things worth remembering

Software changes. These don't:

1. **Clean your data first.** Everything you build afterwards inherits its problems.
2. **Structure it properly.** Facts and dimensions, one row per record, every time.
3. **Say what it means.** A number without a comparison and a recommendation changes nothing.
